# 📋 Action Plan Data Pipeline (API + CSV → Delta Lake)

This module is responsible for ingesting, standardizing, and merging action plan data from multiple sources into a unified, analytics-ready dataset.

---

## 📌 Overview

This pipeline integrates data from:

- A REST API (external service)
- CSV files (historical or manual inputs)

It performs schema harmonization, data cleaning, and business rule standardization before storing the results in a structured Delta table.

---

## 🏗️ Pipeline Architecture
```
REST API
↓
CSV File
↓              
Data Extraction (API / CSV)
↓
Schema Normalization
↓
Data Cleaning & Standardization
↓
Data Unification (unionByName)
↓
Business Rules Application
↓
Delta Table (Action Plan Layer)
```

--- 
## ⚙️ Key Features

### ✅ Multi-Source Ingestion
- Combines data from API and CSV
- Ensures compatibility between heterogeneous schemas

---

### ✅ Schema Harmonization
- Standardizes column names across sources
- Uses dynamic mapping for flexible ingestion

---

### ✅ Data Cleaning
- Trims and sanitizes text fields
- Handles inconsistent values (nulls, blanks, special characters)

---

### ✅ Date Handling
- Converts multiple input formats into consistent timestamp fields
- Uses fallback strategies to avoid pipeline failures

---

### ✅ Safe Data Merging
- Uses `unionByName` to combine datasets
- Handles missing columns gracefully

---

### ✅ Data Lineage
- Tracks source of data (`API` or `CSV`)
- Enables traceability for auditing and debugging

---

## 🧠 Processing Highlights

- Dynamic column mapping using normalized metadata
- Robust handling of semi-structured data
- Flexible schema alignment for evolving datasets
- Resilient ingestion pipeline (partial failures do not break execution)
